## 根因诊断

### 问题 1：节点每次 Step 重新生成 / 完全不显示
**根因（三层叠加）：**
1. `VisualizerView.jsx` 中 `GraphCanvas` 受 `structures.some(s => graphTypes.has(s.type))` 条件渲染控制。当 snap 为 null（程序未运行）时，`Visualizer.jsx` 渲染 `PlaceholderGrid`，`VisualizerView` 不挂载，导致 `ReactFlowProvider` 被卸载再重装，所有内部视口状态丢失。
2. `GraphStore.updateGraph()` 在 `objectMap.size === 0` 时直接返回，即非 tree/linkedlist 类型永远不进入 GraphStore，画布上什么都看不到。
3. `VisualizerView` 既从 `Visualizer.jsx` 接收 `variables/prevVariables` 又内部再调用一次 `buildVisualizerState`，但渲染路径只走 CSS block，React Flow 画布只挂载不渲染节点。

### 问题 2：普通数组、队列、栈、对象、原始值无画布呈现
**根因：** GraphStore 只处理 tree/linkedlist 指针结构，其他类型被丢给 CSS block 渲染，没有任何持久性节点概念。

## 数据流梳理（修复后）

```
用户代码 → InterpreterController.step()
  → Adapter.capture() → VisualState snapshot
  → TimelineStore.pushSnapshot()

Visualizer.jsx:
  snap 变化 → buildVisualizerState(curr, prev)
  → structures[] (含 matrix / array / stack / queue / object / primitive / tree / linkedlist)
  → graphStore.updateGraph(structures, stepIndex)

GraphStore:
  tree/linkedlist → pointer nodes（GlassNode），带 next/left/right edges
  其他类型 → card nodes（CardNode），每变量一个持久卡片

VisualizerView（始终挂载）:
  useGraphStore(s => s.nodes/edges)
  → ReactFlow 渲染全部节点（GlassNode + CardNode）
  → 节点持久存在，只更新不删除
```

## 追加修复（同日第二次执行）

### 问题现象
React Flow 容器完全空白（连背景 dots 都不显示），但 Watch 面板有变量数据，执行线指示器正常，说明数据流通但画布未渲染。

### 根因分析

**直接根因：容器高度链断裂 + overflow 裁切**

1. **`VisualizerView` 用 `h-full min-h-0` 在 `overflow-y-auto` 父容器里** — `overflow-y-auto` 容器会为子元素建立新的滚动上下文，`h-full` 在其中解析为 0（无法从滚动容器继承高度），导致 ReactFlow 容器高度为 0，完全不渲染。

2. **`overflow-hidden` 裁切 ReactFlow 绝对定位元素** — `Visualizer.jsx` 外层有 `overflow-hidden`，ReactFlow v11 内部节点用 `position: absolute` + `transform` 定位，被 `overflow: hidden` 直接裁掉，即使容器有高度节点也不可见。

3. **`overflow-y-auto` 包裹 ReactFlow** — ReactFlow 需要一个确定像素高度且不裁切溢出的父容器，`overflow-y-auto` 会同时剪切宽度和高度方向的绝对定位子元素。

### 修复方案

**`Visualizer.jsx`（布局重构）**
- 移除外层 `overflow-hidden`
- 移除 `overflow-y-auto` 对 VisualizerView 的包裹
- 使用 `flex-col h-full`：上部 `shrink-0` 放步骤计数器 / LineCard，中部 `flex-1 min-h-0` 给 VisualizerView，下部 `shrink-0 max-h-40 overflow-y-auto` 放 StackVisual
- ReactFlow 画布获得由 flex 布局决定的确定像素高度

**`VisualizerView.jsx`（容器修复）**
- 外层 wrapper 改为 `height: '100%', minHeight: 260`（从父容器继承高度）
- `GraphCanvasInner` 容器改为 `height: '100%', minHeight: 260, position: 'relative'`，移除 `overflow-hidden`
- 删除动态 `height` px 计算（现在高度由 flex 布局决定）
- `fitView` effect 监听 `nodes`（整个数组）而非 `nodes.length`，确保每次节点数据变化都重新 fit
