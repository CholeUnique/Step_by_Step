2026-04-03 第三次执行（edit3.md 任务）

## 本次目标
修复 step 卡在 AST 节点问题 + 修复 React Flow tree/linkedlist 不渲染问题

## 根因分析
1. **step 卡 AST 节点**：`InterpreterController.step()` 每次只调用一次 `interpreter.step()`，而 js-interpreter 是 AST 粒度，同一行可能有 5~20 个 AST 节点，导致用户按一次 Step 行号不变
2. **treeToFlow id 错位 bug**：`traverse(node.left, ...)` 调用前保存 `leftId = String(idCounter)`，但 traverse 内部会先把根节点的 id assign 给 `idCounter++`，导致 leftId 指向的其实是左子树的根，而 edges 的 target 是错误 id
3. **ReactFlow 不渲染**：`ReactFlow` 在 v11 需要 `ReactFlowProvider` 包裹；同时父容器必须有明确的像素高度（百分比高度不生效）
4. **MarkerType**：原代码用字符串 `'arrowclosed'`，需通过 `MarkerType.ArrowClosed` 枚举（或直接用字符串，但 type 字段格式有误）

## 修复内容

### InterpreterController.js
- 新增 `getCurrentLine(interp)` 函数：从 stateStack 顶部提取当前行号
- 新增 `stepToNextLine()` 函数：循环调用 `interpreter.step()` 直到行号变化或执行结束（内循环上限 2000 AST 步防死循环）
- `step()` 改为调用 `stepToNextLine()`，实现 statement-level stepping

### VisualizerAdapter.js
- **修复 treeToFlow id 错位**：改为先分配当前节点 id（`idCounter++`），再用 `String(idCounter)` peek 子节点将被分配的 id，然后才调用 traverse 递归——确保 edge target 与实际子节点 id 一致
- 新增节点 type 字段：`glassNode`（普通节点）/ `nullNode`（null 哨兵）
- listToFlow 补充 null 哨兵节点 + `next` edge label
- treeToFlow edges 补充 `left`/`right` label

### VisualizerView.jsx
- 新增 `GlassNode` 自定义节点：Glassmorphism 样式（毛玻璃背景、白色边框、monospace 字体）
- 新增 `NullNode` 自定义节点：虚线边框 + ∅ 符号
- 新增 `FlowCanvas` 共享组件：封装 `ReactFlowProvider` + 固定像素高度容器 + `Background` 点状网格
- `LinkedListBlock` / `TreeBlock` 改用 `FlowCanvas`
- tree 高度根据深度动态计算（`depth * 100 + 60`）

2026-04-03 第五次执行（edit4.md Bug 修复）

## 问题现象
React Flow 画布背景网格可见，但完全空白，节点和连线均不显示。

## 根因分析（三处叠加）

### 1. fitView 时序错误
`ReactFlow` 的 `fitView` prop 只在组件**初始挂载**时执行一次（此时 nodes=[]，无节点可 fit）。之后 GraphStore 更新 nodes，节点坐标被写入但视口没有重新对准，导致节点"存在但不可见"。

### 2. hasGraph 判断方向错误
`VisualizerView` 用 `structures.some(s => graphTypes.has(s.type))` 来决定是否渲染 `GraphCanvas`。这意味着：GraphStore 里可能已有 nodes，但如果当前 snap 对应的 structures 里没有 tree/linkedlist，GraphCanvas 就不渲染。反之，nodes 为空时 GraphCanvas 照样渲染，fitView 无节点可 fit。

### 3. 位置回退值为 {x:0, y:0}
当 `assignPositions` 的 posMap 因某种原因没有覆盖到某个 id 时，位置回退为 `{x:0, y:0}`。在某些 ReactFlow 版本中，所有节点堆在原点，fitView 会将视口缩得极小，节点不可见。

## 修复内容

### VisualizerView.jsx
- 新增 `GraphCanvasInner` 组件（在 `ReactFlowProvider` 内部），用 `useReactFlow().fitView()` 替代 prop `fitView`
- `useEffect([nodes])` 监听节点变化，50ms 后调用 `fitView({ padding:0.3, duration:200 })`——确保每次新节点加入后视口自动调整
- `hasGraph` 判断改为：`graphNodes.length > 0 || structures.some(...)` —— 以 graphStore 为权威，nodes 非空时始终显示画布
- 订阅 `useGraphStore(s => s.nodes)` 获取 `graphNodes`

### graphStore.js
- 新节点位置回退值从 `{x:0, y:0}` 改为 `{ x: fallbackX, y: 20 }`，fallbackX 随节点数递增（每节点 +150px），确保回退节点不会堆叠在原点
